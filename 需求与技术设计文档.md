# 个人博客 + 自定义AI智能体项目 — 需求与技术设计文档

**文档说明**：本文档整合了项目需求分析与技术选型设计，涵盖功能需求、非功能需求、项目边界、技术架构、技术栈选型等内容，可直接作为开发依据。

---

# 第一部分：需求分析

## 一、项目总体概述

### 1.1 项目背景

在互联网学习和自我提升的场景下，需要一个专属个人的内容展示平台，同时伴随大模型技术普及，个人对智能辅助创作、智能问答的需求日益增长。本博客既用于记录学习笔记、技术心得、生活随笔，替代第三方博客平台（CSDN、掘金、知乎）的限制，拥有完全自主的内容版权、页面样式和数据控制权；又能通过自定义智能体，借助大模型能力实现创作辅助、问题解答、建议生成等智能交互，打造"内容展示+智能辅助"一体化的专属个人线上主页。

### 1.2 项目目标

- 实现**轻量化、高颜值、稳定可用**的个人独立博客，新增智能体核心能力
- 支持博主自主后台管理，无需代码即可发布、修改文章，同时可配置专属智能体
- 支持游客在线浏览、搜索，注册用户可评论互动、发表文章、使用 AI 智能体
- 适配电脑、手机端访问，页面自适应
- 支持云端部署，外网可直接访问，长期稳定运行
- 智能体支持博主自定义大模型密钥，灵活对接主流大模型，实现个性化智能交互

### 1.3 适用用户

- **管理员（博主）**：拥有后台全部权限，负责内容发布、审核、管理，以及系统默认智能体的创建、配置、管理
- **注册用户**：注册登录后可以发表文章、评论互动、创建个人智能体、与智能体对话
- **普通游客**：未登录访问，可浏览文章、搜索，不能评论和使用智能体

## 二、功能性需求

### 2.1 前台用户功能（游客可见）

#### 2.1.1 首页功能
- 展示博客首页轮播/简介、博主个人信息、头像、签名
- 分页展示最新发布的文章列表，显示文章标题、简介、发布时间、分类、阅读量
- 展示热门文章、最新评论、文章分类、标签归档
- 展示博主配置的智能体入口（如智能助手卡片），支持游客快速进入智能体交互页面

#### 2.1.2 文章浏览功能
- 文章详情页完整展示正文内容，支持Markdown格式渲染（代码高亮、图片、表格）
- 自动统计文章阅读量，每次独立访问累加
- 显示文章所属分类、标签、发布时间
- 支持上一篇、下一篇文章快速切换
- 可选在文章详情页嵌入智能体入口，支持游客针对当前文章向智能体提问

#### 2.1.3 检索筛选功能
- 关键词全文搜索文章标题、内容
- 按分类筛选对应全部文章
- 按标签筛选对应全部文章

#### 2.1.4 评论互动功能
- 注册用户登录后即可发表评论
- 评论自动显示用户头像和昵称，无需手动填写
- 文章作者评论时显示「作者」标识
- 支持评论展示、按时间排序、分页
- 博主可在后台审核/删除违规评论

#### 2.1.5 博客声明页面
- 展示博客声明内容，支持 Markdown 渲染
- 后台可编辑修改声明内容
- 默认声明："此博客服务于每一个热爱交流的技术选手"

#### 2.1.6 智能体交互功能
- 未登录状态下智能助手界面提示登录后使用
- 登录后可访问系统默认智能体和自建智能体
- 首页侧边栏智能体选择器，可切换不同智能体对话，对话记录独立保存
- 用户可自建智能体：选择大模型厂商、填入 API 密钥、选择模型、配置性格
- 不同用户的对话记录完全隔离
- 智能体回复速度适配普通网络环境，保证交互体验

#### 2.1.7 用户创作功能
- 注册用户可发表文章，支持 Markdown 编辑、分类标签选择、草稿/发布状态
- 用户文章列表管理（我的文章），支持编辑、删除
- 文章详情页展示作者头像和昵称
- 用户个人资料页：头像上传、昵称、邮箱、简介编辑、修改密码

### 2.2 后台管理功能（仅管理员）

#### 2.2.1 登录权限功能
- 管理员账号密码登录后台，权限隔离，游客无法访问后台
- 登录状态保持，支持退出登录
- 简单防暴力登录，错误提示拦截

#### 2.2.2 文章管理功能（核心）
- 新增文章：支持Markdown编辑器，图文混排、代码编辑
- 编辑文章：修改已发布文章内容、标题、分类、标签
- 删除文章：支持单篇/批量删除
- 文章状态控制：草稿、已发布（草稿前台不可见）

#### 2.2.3 分类管理功能
- 新增、编辑、删除文章分类
- 分类唯一，避免重复，关联对应文章

#### 2.2.4 标签管理功能
- 自定义新增、删除文章标签
- 一篇文章可绑定多个标签，一个标签对应多篇文章

#### 2.2.5 评论管理功能
- 查看所有用户评论
- 删除违规、无效评论
- 审核评论（可选开启）

#### 2.2.6 个人信息与站点设置
- 修改博主头像、昵称、简介
- 修改博客站点名称、网站简介、底部版权信息
- 修改管理员登录密码

#### 2.2.7 智能体管理功能
- 系统默认智能体创建：配置名称、能力描述；选择大模型提供商（OpenAI/DeepSeek/通义千问/智谱GLM/Moonshot/豆包）；填入 API 密钥（加密存储）；选择推荐模型；配置模型性格（热情/温柔/冷酷）
- 智能体启用/禁用切换
- 系统默认智能体对所有用户可见可用，但用户不可修改其厂商和模型
- 用户自建智能体由用户独立管理

## 三、非功能性需求

### 3.1 性能需求
- 页面加载速度快，普通网络环境下2秒内完成加载
- 支持日常正常访问并发，满足个人博客使用场景
- 文章数据、用户数据、智能体配置数据持久化存储，不丢失
- 智能体对话响应速度快，普通网络环境下5秒内返回回复（受大模型接口限制除外）
- 大模型API调用异常时，智能体界面给出友好提示，不影响博客整体运行

### 3.2 兼容性需求
- 适配Chrome、Edge、Firefox主流浏览器
- 自适应PC端、手机移动端屏幕
- 智能体交互界面适配不同屏幕尺寸

### 3.3 安全性需求
- 后台权限隔离，无登录权限无法访问管理页面
- 密码、大模型API密钥加密存储，不明文保存
- 基础防SQL注入、非法参数拦截
- 智能体对话内容过滤敏感词，拦截违规提问/回复
- 限制单IP/用户对智能体的调用频率，防止恶意消耗API额度

### 3.4 可维护性需求
- 代码结构清晰，分层规范，便于后续新增功能
- 数据结构设计合理，方便后期迭代优化
- 部署简单，一键启动，维护成本低
- 智能体配置界面提供清晰的参数说明

## 四、项目边界（不做功能）

- 不做复杂的会员、点赞、收藏、积分功能
- 不做复杂的权限分级、多角色管理（仅管理员+注册用户两级）
- 不做付费、广告、流量统计等商业化功能
- 智能体模块不做自定义模型训练、本地大模型部署，仅对接公开大模型API
- 不做智能体的多版本管理、复杂的对话流程编排

# 第二部分：技术设计

## 五、项目整体架构

前后端分离架构，无强耦合、无复杂中间件，未来可直接 Docker 一键打包部署。

```
前端（用户展示端 + 管理后台 + AI 智能体对话）
        ↓
后端服务（SpringBoot 3）
        ↓
数据层：MySQL 8.0（持久化） + Redis 8.4（缓存/高性能存储）
        ↓
外部服务：主流大模型 API（GPT、文心一言、通义千问、DeepSeek 等）
```

## 七、技术选型原则

1. 无自定义编译、无复杂环境依赖
2. 配置全部外置，不写死代码
3. 环境隔离：开发 / 测试 / 生产无缝切换
4. 无特殊系统依赖，Windows / Linux / Mac 都能跑
5. 未来可直接打包成 Docker 镜像，一行命令启动
6. 日志规范、接口统一、结构清晰

## 八、数据库选型

### 8.1 MySQL 8.0（主存储）

存储必须永久保存的数据：
- 管理员信息
- 文章、分类、标签、评论
- AI 智能体配置（名称、角色、大模型密钥加密存储）

### 8.2 Redis 8.4（高性能缓存）

- 缓存首页文章列表（页面秒开）
- 管理员登录 Token 存储
- AI 智能体对话历史记忆（上下文连续对话）
- 接口限流防刷（保护大模型 API 额度）
- 热点数据缓存（阅读量、访问量）

## 九、AI 智能体模块技术方案

- 后端通过 HTTP 调用第三方大模型 API
- 管理员在后台配置自己的 API Key
- Key AES 加密存储在 MySQL
- 对话历史存在 Redis，实现上下文记忆
- 支持配置：角色、温度、最大回复长度、上下文轮数
- 支持接口限流 + 敏感词过滤

## 十、便于部署的关键设计

1. 所有配置统一放在 application.yml（端口、数据库、Redis、AI 调用超时等）
2. 无环境强依赖，不依赖系统环境变量、不绑定固定路径
3. 日志文件自动输出，便于部署后排查问题
4. 接口统一返回格式，便于前端、部署、监控
5. 全局异常统一处理
6. 支持多环境配置（application-dev.yml / application-prod.yml）

## 十一、开发预期成果

开发完成后，将获得一个完整、可上线、可长期维护的"内容+智能"一体化个人独立博客：
- 外网可访问的个人博客官网，兼具内容展示与智能交互能力
- 完整的前后端功能闭环，包含智能体创建、配置、交互全流程
- 自主可控的文章数据、智能体配置数据，无第三方平台限制
- 智能体支持博主自定义大模型密钥，灵活对接主流大模型
- 可用于简历项目展示、个人技术沉淀、作品集
