# 第一部分：环境配置

## 一、配置文件结构

```
personal_broadcast/
├── .gitignore            # Git 忽略配置
└── backend/
    ├── .env              # 环境变量配置（本地开发，不提交到 Git）
    ├── .env.example      # 环境变量模板（提交到 Git）
    └── src/
        └── main/
            └── resources/
                ├── application.yml          # 公共配置
                ├── application-dev.yml      # 开发环境配置
                └── application-prod.yml     # 生产环境配置
```

## 二、快速开始

### 2.1 复制环境变量模板

```bash
cd backend
cp .env.example .env
```

### 2.2 修改 `.env` 文件

```bash
# 数据库配置
DB_HOST=localhost
DB_PORT=3306
DB_NAME=blog
DB_USERNAME=root
DB_PASSWORD=your_password_here

# Redis 配置
REDIS_HOST=localhost
REDIS_PORT=6379
REDIS_PASSWORD=

# JWT 密钥（生产环境请修改为随机生成的 256 位密钥）
JWT_SECRET=blog-jwt-secret-key-must-be-at-least-256-bits-long-for-hs256

# AES 密钥（必须是 32 字节）
AES_SECRET_KEY=blog-aes-key-256bits-must-be-32bytes!!
```

### 2.3 启动应用

**开发环境**：
```bash
cd backend
mvn spring-boot:run
```

**生产环境**：
```bash
export SPRING_PROFILES_ACTIVE=prod
mvn spring-boot:run

# 或通过命令行参数
mvn spring-boot:run -Dspring-boot.run.profiles=prod
```

## 三、配置说明

### 3.1 环境变量

所有敏感信息通过环境变量注入，格式 `${VAR_NAME:default}`：
- `VAR_NAME`: 环境变量名
- `default`: 默认值（环境变量不存在时使用）

### 3.2 环境差异

| 配置项 | 开发环境 (dev) | 生产环境 (prod) |
|--------|--------------|--------------|
| 数据库初始化 | 自动初始化 | 不初始化 |
| 日志级别 | DEBUG | INFO |
| Redis 连接池 | 8 | 16 |
| SQL 日志 | 输出 | 不输出 |
| 日志输出 | 控制台 | 文件 |

### 3.3 项目默认配置值

后端 `application-dev.yml` 中的默认值：

```yaml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/blog?...
    username: root
    password: root
  data:
    redis:
      host: localhost
      port: 6379
      password:
```

## 四、安全建议

1. **生产环境必须修改默认密钥**
   - JWT_SECRET：使用随机生成的 256 位密钥
   - AES_SECRET_KEY：使用随机生成的 32 字节密钥

2. **不要将 `.env` 文件提交到 Git**（已在 `.gitignore` 中忽略）

3. **使用环境变量管理敏感信息**
   ```bash
   # Linux/Mac
   export DB_PASSWORD=your_password
   mvn spring-boot:run

   # Windows PowerShell
   $env:DB_PASSWORD="your_password"
   mvn spring-boot:run
   ```

# 第二部分：MySQL 与 Redis 运行指南

## 五、环境要求

| 组件  | 版本  | 端口  |
|-------|-------|-------|
| MySQL | 8.0+  | 3306  |
| Redis | 8.4+  | 6379  |

## 六、Windows 环境

### 6.1 MySQL 8.0

**方式 A：官方安装包**
1. 下载 MySQL 8.0 Installer
2. 安装时选择 `Server only`，端口保持默认 3306
3. 设置 root 密码（与项目配置一致）

**方式 B：免安装压缩包**
```powershell
# 初始化（生成 data 目录和临时密码）
$MYSQL_HOME/bin/mysqld --initialize-insecure --console

# 安装为 Windows 服务
$MYSQL_HOME/bin/mysqld --install MySQL80

# 启动服务
net start MySQL80

# 设置 root 密码
$MYSQL_HOME/bin/mysql -u root --skip-password
ALTER USER 'root'@'localhost' IDENTIFIED BY 'root';
FLUSH PRIVILEGES;
```

**初始化数据库**：
```powershell
$MYSQL_HOME/bin/mysql -u root -proot < blog_init.sql
```

### 6.2 Redis 8.4

**方式 A：Docker Desktop（推荐）**
```powershell
docker run -d --name redis -p 6379:6379 redis:8.4-alpine
docker exec -it redis redis-cli PING
# 预期返回: PONG
```

**方式 B：Windows 兼容版本**
1. 下载 [Redis for Windows](https://github.com/tporadowski/redis/releases)
2. 解压后双击 `redis-server.exe` 启动

## 九、Docker 一键启动（所有平台通用）

```yaml
version: '3.8'
services:
  mysql:
    image: mysql:8.0
    container_name: blog-mysql
    restart: always
    ports:
      - "3306:3306"
    environment:
      MYSQL_ROOT_PASSWORD: root
      MYSQL_DATABASE: blog
    volumes:
      - ./blog_init.sql:/docker-entrypoint-initdb.d/init.sql
      - mysql_data:/var/lib/mysql

  redis:
    image: redis:8.4-alpine
    container_name: blog-redis
    restart: always
    ports:
      - "6379:6379"
    volumes:
      - redis_data:/data

volumes:
  mysql_data:
  redis_data:
```

```bash
# 启动所有服务
docker compose up -d

# 验证 MySQL
docker exec -it blog-mysql mysql -u root -proot -e "USE blog; SHOW TABLES;"

# 验证 Redis
docker exec -it blog-redis redis-cli PING

# 停止服务
docker compose down
```

## 十、常见问题

| 问题 | 解决方法 |
|------|----------|
| MySQL 端口被占用 | `netstat -ano \| findstr 3306` 查看占用进程 |
| Redis 连接被拒 | 确认 redis-server 已启动，检查是否绑定了 127.0.0.1 |
| 数据库不存在 | `mysql -u root -proot -e "CREATE DATABASE blog DEFAULT CHARSET utf8mb4;"` |
| 表已存在 | `blog_init.sql` 使用 `DROP TABLE IF EXISTS`，重复执行会重建 |
| Docker 连接超时 | 检查防火墙是否放行 3306/6379 端口 |
| 中文乱码 | 确认数据库字符集为 `utf8mb4`，连接 URL 含 `characterEncoding=utf8mb4` |

### 配置故障排查

**数据库连接失败**：检查 `backend/.env` 中的配置
**Redis 连接失败**：检查 Redis 是否运行
**密钥长度错误**：JWT_SECRET 至少 256 位（32 字节），AES_SECRET_KEY 必须 32 字节

curl http://localhost:8080/api/v1/site/config

## 十六、快速启动脚本

### Windows（保存为 `start.bat`）
```bat
@echo off
start "Backend" cmd /c "cd backend && mvn spring-boot:run"
start "Frontend" cmd /c "cd frontend && npm run dev"
echo 后端: http://localhost:8080
echo 前端: http://localhost:3000
```

### macOS / Linux（保存为 `start.sh`）
```bash
#!/bin/bash
cd backend && mvn spring-boot:run &
cd frontend && npm run dev &
echo "后端: http://localhost:8080"
echo "前端: http://localhost:3000"
wait
```

# 第四部分：配置重构记录

## 十七、配置管理优化

本项目已完成配置文件重构，实现了敏感信息保护和多环境配置分离：

### 重构内容
- 创建 `.env` 存储敏感信息（数据库密码、密钥等），`.env.example` 作为模板可安全提交到 Git
- `application.yml` 只保留公共配置
- `application-dev.yml` 只保留开发环境特有配置
- `application-prod.yml` 只保留生产环境特有配置
- 所有配置使用环境变量格式 `${VAR_NAME:default}`

### 已保护的敏感信息
- 数据库密码
- JWT 密钥
- AES 密钥
- Redis 密码

### 生产环境注意事项
1. 必须修改 `JWT_SECRET`（随机生成 256 位密钥）
2. 必须修改 `AES_SECRET_KEY`（随机生成 32 字节密钥）
3. 必须修改 `DB_PASSWORD`（使用强密码）
4. 不要提交 `.env` 到 Git
5. 生产环境的 `.env` 文件应单独备份
